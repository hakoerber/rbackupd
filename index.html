description

rbackupd is a backup program for linux using rsync. It creates "snapshots" of the filesystem that contain all files and directories of that filesystem at the time of the snapshot. To reduce the required disk space, files that have not changed since the last snapshot are hardlinked into the new snapshot.

You can specify multiple intervals in which snapshots will be created, and a maximum number of snapshots that should be kept for every interval. Intervals can be given in a cron-like format, which means you have great flexibility in specifying these intervals.

Thanks to rsync, rbackupd will preserve hardlinks, special files such as named sockets and fifos, device files, permissions, ownership, modification times, ACLs and extended attributes.

As the snapshots are neither compressed nor encrypted by rbackupd, every user can access all files owned by him without requiring root privileges.

installation

First, you need to get the source code by cloning the repository. To get a folder named rbackupd in the current directory, run$ git clone http://github.com/whatevsz/rbackupd.gitNow enter the folder with $ cd rbackupdTo install, just run the following command with root privileges:# python setup.py install

Now look into the template configuration file located at /etc/rbackupd/rbackupd.conf. Every options has a short description, so an initial setup should be easy. You should generally be fine with leaving everything as it is, and just change the sources and destination keys to the desired paths. For example, if you want to back up the entire /home/ and /etc/ directories to a device mounted at /mnt/backup, set them to sources = /home/, /etc/
destination = /mnt/backup
